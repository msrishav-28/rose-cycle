# API_SPEC.md - RoseCycle API Contract

All backend behavior in backend/app/api/v1/endpoints/ must match this contract.
Document changes first or in the same PR as implementation.

## 1. Base URLs
- Local backend: http://127.0.0.1:8000
- API prefix: /api/v1

## 2. Authentication
Protected HTTP endpoints require:
- Authorization: Bearer <clerk_jwt>

Notes:
- In local dev, backend can fall back to a placeholder user id if Clerk config is missing.
- This fallback is development-only behavior and must not be treated as production auth.

## 3. Health Endpoints

### GET /
Description: Root liveness message.
Response 200:
{
  "message": "RoseCycle API is live."
}

### GET /health
Description: Health probe endpoint.
Response 200:
{
  "status": "healthy"
}

## 4. Daily Logs

### POST /api/v1/logs
Auth: required
Description: Create or update a daily log for the authenticated user by date.

Request body:
{
  "date": "YYYY-MM-DD",
  "flow": "None | Light | Medium | Heavy | V. Heavy | null",
  "mood": "string | null",
  "pain_level": 1-10,
  "notes": "string | null"
}

Response 200 or 201 (current implementation returns model object):
{
  "id": 1,
  "user_id": 1,
  "date": "2026-04-12",
  "flow": "Medium",
  "mood": "Calm",
  "pain_level": 4,
  "notes": "string"
}

Behavior notes:
- If a log exists for same user and date, endpoint updates it.
- Otherwise, endpoint creates a new row.
- Validation enforces pain_level in range 1-10 when provided.

### GET /api/v1/logs
Auth: required
Description: Return all daily logs for the authenticated user.

Response 200:
[
  {
    "id": 1,
    "user_id": 1,
    "date": "2026-04-12",
    "flow": "Light",
    "mood": "Tired",
    "pain_level": 3,
    "notes": "string"
  }
]

Behavior notes:
- Logs are returned in descending date order (newest first).

## 5. Cycle Insights

### GET /api/v1/cycle/insights
Auth: required
Description: Return cycle summary values for dashboard insight cards.

Current response shape:
{
  "current_day": 11,
  "phase_name": "Follicular Phase",
  "next_period_in_days": 17,
  "proactive_insight": "Your most frequently logged mood recently is 'Calm'..."
}

Contract requirement:
- Keep field names stable until versioned.
- Field values are computed from stored daily logs and inferred period-flow patterns.

## 6. Chat Stream

### WS /api/v1/chat/stream
Auth: token required when Clerk is configured; dev fallback remains when Clerk is not configured
Description: Bidirectional chat stream to LangGraph agent.

Connection format:
- ws://127.0.0.1:8000/api/v1/chat/stream?token=<clerk_jwt>

Client -> server payload:
{
  "message": "string"
}

Server -> client payload:
{
  "type": "message",
  "content": "string"
}

Behavior notes:
- Server stores conversation memory per websocket connection.
- Response currently generated by LangGraph graph using configured model path.
- If provider keys are missing, backend uses deterministic placeholder fallback.

## 7. Error Behavior (Current + Expected)
Current implementation raises FastAPI default errors for validation/auth failures.
Target standardization (recommended):
{
  "error": {
    "code": "UNAUTHORIZED | VALIDATION_ERROR | INTERNAL_ERROR",
    "message": "human readable"
  }
}
